## 1. Control Systems: Principles and Mathematics

### 1.1 Feedback Control — The Core Principle

**Negative feedback loop**: System generates output \\(y(t)\\); desired value is reference \\(r(t)\\); error is:

\\[e(t) = r(t) - y(t)\\]

Controller produces control input \\(u(t)\\) proportional to error, driving the system to reduce \\(e(t)\\) toward zero. Negative feedback (subtracting actual from desired) stabilizes behavior and rejects disturbances. This is the central principle of all automatic control.

**Open-loop vs. closed-loop**:
- **Open-loop**: No measurement of output; control based on model alone. Example: timed irrigation. Susceptible to disturbances.
- **Closed-loop (feedback)**: Output measured and fed back to correct error. Example: cruise control. Robust to disturbances but can oscillate if not designed carefully.

### 1.2 State-Space Representation

The most general linear system representation:

\\[\\dot{x}(t) = Ax(t) + Bu(t)\\]
\\[y(t) = Cx(t) + Du(t)\\]

where:
- \\(x \\in \\mathbb{R}^n\\) — state vector (internal system memory)
- \\(u \\in \\mathbb{R}^m\\) — control input vector
- \\(y \\in \\mathbb{R}^p\\) — measured output vector
- \\(A \\in \\mathbb{R}^{n \\times n}\\) — system matrix (dynamics)
- \\(B \\in \\mathbb{R}^{n \\times m}\\) — input matrix
- \\(C \\in \\mathbb{R}^{p \\times n}\\) — output matrix
- \\(D \\in \\mathbb{R}^{p \\times m}\\) — feedthrough matrix

**Discrete-time version** (for digital controllers with sampling period \\(T_s\\)):

\\[x[k+1] = A_d x[k] + B_d u[k]\\]
\\[y[k] = Cx[k] + Du[k]\\]

where \\(A_d = e^{AT_s}\\), \\(B_d = \\int_0^{T_s} e^{A\\tau} B d\\tau\\). Digital control requires sampling fast enough to capture system dynamics (Shannon-Nyquist: \\(f_s > 2 f_{max}\\)).

### 1.3 Transfer Function (Laplace Domain)

For linear time-invariant systems, the transfer function:

\\[G(s) = \\frac{Y(s)}{U(s)} = C(sI - A)^{-1}B + D\\]

captures input-output relationships in the frequency domain. Key tools:
- **Bode plots**: Magnitude and phase vs. frequency \\(\\omega\\). Used for stability margins and controller design.
- **Root locus**: How closed-loop poles move as controller gain varies. Used to design stable controllers.
- **Nyquist plot**: Polar plot of \\(G(j\\omega)\\); encircles \\(-1\\) point ⟺ closed-loop system unstable.

### 1.4 PID Control

The dominant industrial controller (95% of industrial controllers worldwide):

\\[u(t) = K_p e(t) + K_i \\int_0^t e(\\tau)d\\tau + K_d \\frac{de(t)}{dt}\\]

**Each term's role**:
- **Proportional** \\(K_p e(t)\\): Immediate correction proportional to current error. Faster response, but steady-state error remains.
- **Integral** \\(K_i \\int e dt\\): Accumulates past error; eliminates steady-state error. Risk: integral windup during saturation.
- **Derivative** \\(K_d \\dot{e}\\): Anticipates future error from rate of change; dampens overshoot. Amplifies measurement noise.

**PID Tuning Methods**:

| Method | Approach | Best For |
|---|---|---|
| **Ziegler-Nichols** | Induce oscillation at stability boundary; use empirical formulas | Quick starting point; often aggressive |
| **Cohen-Coon** | Step response method using dead time and time constant | Processes with significant dead time |
| **Loop shaping** | Design frequency response to meet bandwidth/margin specs | Systematic, model-based design |
| **Auto-tuning** | Built-in algorithms in modern PLCs (relay feedback test) | Industrial deployment without models |

**Anti-windup**: When actuator saturates, integral term continues accumulating error (windup), causing large overshoot when saturation ends. Solutions: back-calculation, conditional integration, output clamping.

### 1.5 Stability Analysis

**Lyapunov stability**:
- System is **stable** if trajectories starting near equilibrium remain near it.
- System is **asymptotically stable** if additionally trajectories converge to equilibrium.
- **Lyapunov direct method**: If there exists \\(V(x) > 0\\) for all \\(x \\neq 0\\) with \\(\\dot{V}(x) < 0\\), the system is asymptotically stable. No need to solve differential equations.

**Linear system stability**:
- Stable ⟺ all eigenvalues of \\(A\\) have negative real parts.
- **Routh-Hurwitz criterion**: Algebraic test for polynomial roots; determines stability without computing eigenvalues.

**Frequency domain margins**:
- **Gain margin**: How much gain can increase before instability. Target: >6 dB.
- **Phase margin**: How much phase lag before instability. Target: >45°.

### 1.6 Observer Design — Estimating Hidden States

Most systems have states not directly measurable (e.g., velocity from position sensor only). **Observers** (state estimators) reconstruct the state:

**Luenberger observer**:
\\[\\dot{\\hat{x}} = A\\hat{x} + Bu + L(y - C\\hat{x})\\]

where \\(L\\) is the observer gain matrix chosen to make \\((A - LC)\\) Hurwitz (stable). Design: pole placement for observer dynamics (typically 2-5× faster than plant).

**Kalman filter** (optimal observer for stochastic systems):
- Assumes process noise \\(w \\sim \\mathcal{N}(0, Q)\\) and measurement noise \\(v \\sim \\mathcal{N}(0, R)\\).
- Computes optimal gain \\(K_k\\) minimizing estimation error covariance.
- **Prediction**: \\(\\hat{x}_{k|k-1} = A\\hat{x}_{k-1} + Bu\\)
- **Update**: \\(\\hat{x}_{k} = \\hat{x}_{k|k-1} + K_k(y_k - C\\hat{x}_{k|k-1})\\)
- Kalman gain: \\(K_k = P_{k|k-1} C^T (C P_{k|k-1} C^T + R)^{-1}\\)
- Applications: SLAM, sensor fusion in autonomous vehicles, INS/GPS integration.

### 1.7 Optimal Control — LQR and MPC

**Linear Quadratic Regulator (LQR)**:

Minimize \\(J = \\int_0^\\infty (x^T Q x + u^T R u) dt\\)

where \\(Q \\geq 0\\) penalizes state deviation and \\(R > 0\\) penalizes control effort. Solved via the algebraic Riccati equation:

\\[A^T P + PA - PBR^{-1}B^T P + Q = 0\\]

Optimal gain: \\(K = R^{-1}B^T P\\); Control law: \\(u = -Kx\\).

**Model Predictive Control (MPC)**:

At each time step, solve:
\\[\\min_{u_0, ..., u_{N-1}} \\sum_{k=0}^{N-1} (x_k^T Q x_k + u_k^T R u_k) + x_N^T P x_N\\]

subject to: state/input constraints (actuator limits, safety margins). Only the first control input is applied; replan at next step. MPC is the gold standard for constrained multi-variable industrial control.

**Adaptive Control**:
Parameters \\(K_p, K_i, K_d\\) or model parameters update online using recursive least squares or MIT rule. Maintains performance as system dynamics change (e.g., load changes, aging).

## 2. Automata Theory and State Machines

### 2.1 Finite State Machines (FSM)

Abstract computational model: \\((S, \\Sigma, \\delta, s_0, F)\\)
- \\(S\\): finite set of states
- \\(\\Sigma\\): input alphabet (events/signals)
- \\(\\delta: S \\times \\Sigma \\rightarrow S\\): transition function
- \\(s_0\\): initial state
- \\(F\\): accepting/final states

**Relevance to automation**: Every PLC program is fundamentally an FSM. Ladder logic, structured text, and Sequential Function Charts (SFC) all compile to state machine semantics. Elevators, traffic lights, and robot task sequences are FSMs.

**Hierarchical state machines** (Harel statecharts): States contain substates; enables modular design of complex control logic without state explosion.

### 2.2 Timed Automata

Extension of FSM with real-valued clocks—essential for real-time systems where timing constraints matter (IEC 61131-3 PLC scan cycle, CAN bus scheduling). Enables formal verification of timing properties using UPPAAL model checker.

### 2.3 Turing Completeness and PLCs

PLCs and microcontrollers are Turing-complete: they can compute any computable function. This means PLC programs can have unbounded complexity and cannot be universally verified by static analysis alone—a key reason why runtime monitoring and testing remain essential.

## 3. Cybernetics and Feedback Principles

### 3.1 Norbert Wiener's Unifying Framework

Wiener's cybernetics (1948) unified:
- Control engineering (feedback, stability)
- Information theory (Shannon entropy, signal transmission)
- Neuroscience (neural feedback loops, homeostasis)
- Organizational theory (management as information processing)

Core thesis: Goal-directed behavior through feedback is universal—applicable to machines, organisms, and organizations equally.

### 3.2 Homeostasis as Automation Model

Biological systems maintain internal equilibrium despite external perturbations:
- Body temperature: Thermoregulation via sweating/shivering (negative feedback on temperature error)
- Blood glucose: Insulin/glucagon secretion (negative feedback on glucose level)

Industrial parallels:
- Temperature control in chemical reactors
- Pressure regulation in hydraulic systems
- Inventory level control in supply chains

### 3.3 Negative vs. Positive Feedback

| Type | Effect | Stability | Industrial Example |
|---|---|---|---|
| **Negative** | Error reduces over time | Stabilizing | Watt's governor, cruise control, PID |
| **Positive** | Error amplifies | Destabilizing | Audio feedback screech, thermodynamic runaway |
| **Positive (useful)** | Amplifies desired signal | Context-dependent | Regenerative braking energy capture, Schmitt trigger hysteresis |

## 4. Reinforcement Learning as Control Theory

### 4.1 Markov Decision Processes (MDP)

Formal framework for RL: \\((S, A, P, R, \\gamma)\\)
- \\(S\\): state space
- \\(A\\): action space
- \\(P(s'|s,a)\\): transition probability
- \\(R(s,a)\\): reward function
- \\(\\gamma \\in [0,1)\\): discount factor

**Bellman equation** (optimal value function):

\\[V^*(s) = \\max_a \\left[ R(s,a) + \\gamma \\sum_{s'} P(s'|s,a) V^*(s') \\right]\\]

Policy gradient methods (PPO, SAC) optimize the expected return \\(\\mathbb{E}[\\sum_t \\gamma^t r_t]\\) directly using gradient ascent.

### 4.2 Connection to Classical Control

| Classical Control | Reinforcement Learning |
|---|---|
| Plant model (state-space) | Environment (MDP) |
| Controller (PID, LQR) | Policy \\(\\pi(a|s)\\) |
| Setpoint tracking | Reward maximization |
| Stability analysis (Lyapunov) | Safe RL (CBF, CMDPs) |
| Model-based (MPC) | Model-based RL (Dyna, DreamerV3) |
| System identification | System identification + RL |

**Key difference**: Classical control requires a model (or can design one). RL can learn from interaction without a model (model-free RL). RL scales to high-dimensional state/action spaces; classical methods struggle with this.

### 4.3 Safe RL (Constrained MDPs)

Constrained MDP extends standard MDP with safety constraints:

\\[\\max_\\pi \\mathbb{E}\\left[\\sum_t \\gamma^t r_t\\right] \\quad \\text{subject to} \\quad \\mathbb{E}\\left[\\sum_t \\gamma^t c_t\\right] \\leq d\\]

where \\(c_t\\) is a constraint cost and \\(d\\) is the constraint threshold. Lagrangian relaxation converts this to an unconstrained dual problem.

## 5. Formal Verification

**Formal methods** prove that a system satisfies a specification for ALL possible inputs and behaviors—contrasting with testing (which checks finite samples).

**Temporal logic specifications**: Express properties over time:
- **LTL (Linear Temporal Logic)**: "The alarm is always eventually reset": \\(\\Box \\Diamond \\text{reset}\\)
- **CTL (Computation Tree Logic)**: Branches over possible futures; "there exists a path where the system recovers": \\(EF \\text{safe}\\)

**Model checking** (SPIN, UPPAAL, NuSMV): Exhaustively explores system state space to verify LTL/CTL properties. Scales to ~10⁸ states with BDD-based representations. Applied to PLC programs, communication protocols, and safety interlocks.

**Theorem proving** (Coq, Isabelle, HOL4): Mathematical proofs of correctness; not automated but provides strongest guarantees. Used for safety-critical avionics software (Airbus A380 certified components).

**Limitations**:
- Real-world system complexity (continuous dynamics, nonlinearity) resists complete symbolic modeling.
- Neural network verification: Active research area (see [Research Frontiers §3](../4_The_Horizons/4.1_Research_Frontiers.md)).
- State explosion: Verification complexity often exponential in system size.
