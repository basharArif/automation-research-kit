## 1. Industrial Automation KPIs

### 1.1 Overall Equipment Effectiveness (OEE)

The gold-standard manufacturing efficiency metric. OEE combines three factors:

\\[OEE = Availability \\times Performance \\times Quality\\]

**Component formulas**:

| Component | Formula | What it measures |
|---|---|---|
| Availability | \\(\\frac{Run\\ Time}{Planned\\ Production\\ Time}\\) | Uptime vs. planned time (excludes planned maintenance) |
| Performance | \\(\\frac{Ideal\\ Cycle\\ Time \\times Total\\ Count}{Run\\ Time}\\) | Actual speed vs. theoretical maximum speed |
| Quality | \\(\\frac{Good\\ Count}{Total\\ Count}\\) | First-pass yield (defect-free output fraction) |

**Industry benchmarks**:

| OEE Score | Classification | Notes |
|---|---|---|
| **85%+** | World-class | Achieved by top Toyota plants; difficult to sustain |
| **65-85%** | Typical good performance | Most modern automated factories |
| **40-65%** | Below average | Common in older or less automated facilities |
| **<40%** | Poor | Often indicates maintenance or scheduling failures |

**Example calculation**: A machine scheduled 8 hours has 30 min downtime (Availability = 87.5%); runs at 90% of rated speed (Performance = 90%); produces 2% defective parts (Quality = 98%).
\\[OEE = 0.875 \\times 0.90 \\times 0.98 = 77.2\\%\\]

**AI impact**: Smart factory implementations report 6–12% OEE increase from AI-driven predictive maintenance and process optimization [McKinsey, 2023].

### 1.2 Mean Time Between Failures (MTBF)

\\[MTBF = \\frac{Total\\ Operating\\ Time}{Number\\ of\\ Failures}\\]

Higher MTBF = more reliable system.

**Industry targets**:
| Equipment Type | Target MTBF |
|---|---|
| Industrial robots (arm) | 50,000–80,000 hours |
| CNC machining centers | 2,000–5,000 hours |
| Conveyor systems | 8,000–15,000 hours |
| PLCs (modern solid-state) | 100,000+ hours (MTBF specified by vendor) |
| Edge AI compute module | 10,000–50,000 hours |

**MTBF limitation**: MTBF is a statistical mean—individual failure times are exponentially distributed. It does not mean the machine will definitely work for MTBF hours before failure.

### 1.3 Mean Time To Repair (MTTR)

\\[MTTR = \\frac{Total\\ Repair\\ Time}{Number\\ of\\ Repairs}\\]

Lower MTTR = faster recovery. Target: **<2 hours** for standard manufacturing equipment.

**Reduction strategies**:
- Predictive maintenance (AI-predicted failures allow pre-staging spare parts)
- Digital work instructions (AR-guided repair procedures)
- Remote diagnostics via OPC UA/cloud (reduce technician travel time)
- Spare parts inventory optimization (ML-based demand forecasting)

**Availability from MTBF and MTTR**:
\\[Availability = \\frac{MTBF}{MTBF + MTTR}\\]

If MTBF = 1000 hours and MTTR = 4 hours: Availability = 99.6%.

### 1.4 Takt Time and Cycle Time

**Takt time** (customer demand rate):
\\[Takt\\ Time = \\frac{Available\\ Production\\ Time}{Customer\\ Demand}\\]

If daily available time = 460 minutes and demand = 230 units/day: Takt = 2 min/unit.

**Cycle time**: Actual time to complete one unit. Must be ≤ Takt time to meet demand.

**First Pass Yield (FPY)**:
\\[FPY = \\frac{Units\\ Passing\\ First\\ Inspection}{Total\\ Units\\ Produced}\\]

World-class: >99%. Six Sigma target: 99.99966% (3.4 defects per million opportunities).

## 2. Robotics & Autonomous Systems KPIs

### 2.1 Task Success Rate

Percentage of tasks completed correctly within specified parameters.

**Benchmark context (2025)**:
| Task Type | State-of-the-Art Success Rate | Method |
|---|---|---|
| CartPole balance (simulated) | 100% | RL (PPO, 50K steps) |
| Safe RL (constrained tasks) | 97.8% | Constrained MDPs [Wachi et al., 2024] |
| Robotic bin picking (novel objects) | 85–92% | RT-2, foundation model policies |
| Dexterous in-hand manipulation | 60–75% | π0, ALOHA [Physical Intelligence, 2024] |
| Warehouse AMR task completion | 98%+ | Amazon Robotics (fixed environment) |

**Measurement**: Run N trials (N ≥ 100 for statistical significance); count successes. Report mean ± 1 standard deviation, and specify exact task and environment.

### 2.2 Latency (Perception → Actuation)

Critical for real-time control. Targets depend on application:

| Application | Latency Target | Rationale |
|---|---|---|
| Edge AI inference (YOLO) | <30ms | Real-time video processing |
| Robot joint control loop | <1ms | Smooth, stable servo control |
| SLAM map update | <100ms | Navigation not critically affected beyond this |
| Autonomous vehicle decision | <100ms | Human reaction time reference baseline |
| Industrial PLC scan cycle | 1–10ms | Process-dependent |
| RPA workflow step | 0.5–5s | Latency vs. accuracy trade-off |

**Measurement**: Use `time.perf_counter()` (Python) or hardware timestamps for sub-millisecond accuracy. Report percentiles (p50, p95, p99) not just mean—tail latency matters in real-time systems.

### 2.3 Sim-to-Real Transfer Gap

\\[Sim2Real\\ Gap = \\frac{Performance_{sim} - Performance_{real}}{Performance_{sim}} \\times 100\\%\\]

**State-of-the-art (2025)**:
- Object manipulation with domain randomization: **<3% gap** [Tobin et al., 2017; replicated in multiple labs]
- Quadruped locomotion (ANYmal, Go1): **<5% gap** on flat terrain [ETH Zürich, 2023]
- Contact-rich manipulation (screws, cloth): **30–50% gap** — active research frontier

**Factors increasing gap**: Contact dynamics, material properties, visual appearance, sensor noise characteristics, actuator dead zones.

### 2.4 Repeatability and Accuracy

For industrial robots:

| Metric | Definition | World-Class Target |
|---|---|---|
| **Repeatability** | Variation returning to same programmed position | ±0.02–0.05mm (UR5e, ABB IRB) |
| **Accuracy** | Absolute deviation from commanded position | ±0.1–0.5mm (depends on calibration) |
| **Path accuracy** | Deviation from commanded Cartesian path | ±0.2mm (welding robots) |
| **Max payload** | Maximum rated load at full reach | Application-specific |

Note: Repeatability ≠ Accuracy. A robot can be highly repeatable (returns to the same spot each time) but inaccurate (that spot is not where commanded). Factory robots are calibrated for high repeatability; absolute accuracy is rarer.

## 3. RPA KPIs

### 3.1 Throughput and Time Savings

| Metric | Formula | Industry Benchmark |
|---|---|---|
| **Time savings** | \\(\\frac{T_{manual} - T_{bot}}{T_{manual}}\\) | 60–90% reduction |
| **Bot throughput** | Transactions/hour per bot | 200–2000 (process-dependent) |
| **FTE equivalent** | FTEs replaced per bot (24/7 operation) | 2–5 FTEs |
| **Process velocity** | Speed increase ratio | 110% improvement [Sutherland, 2023] |

**Example**: Invoice processing manual time = 8 minutes/invoice; bot time = 45 seconds/invoice. Time savings = (8×60 - 45)/(8×60) = 90.6%.

### 3.2 Quality and Accuracy

| Metric | Human Baseline | Bot Benchmark | Notes |
|---|---|---|---|
| **Error rate** | 4–10% | <0.5% | Errors in data entry, calculations |
| **First-run rate** | 75–85% | 90–98% | Transactions completed without exception |
| **SLA compliance** | 80–90% | 95–99%+ | Percentage meeting service level agreements |
| **Exception rate** | — | 2–15% | Transactions requiring human review (process-dependent) |

### 3.3 Financial KPIs

| KPI | Benchmark |
|---|---|
| **Year 1 ROI** | 30–200% |
| **Long-term ROI (3-5 years)** | 200–300% |
| **Cost per transaction (bot)** | $0.05–0.25 vs. $5–15 (manual) |
| **Payback period** | 6–18 months |
| **Implementation cost** | $5,000–$50,000 per automation (process complexity-dependent) |

## 6. Control System Performance Metrics

| Metric | Definition | Measurement | Target |
|---|---|---|---|
| **Rise time** | Time to reach 90% of setpoint | Step response | Depends on bandwidth spec |
| **Settling time** | Time within ±2% of setpoint | Step response | Depends on process dynamics |
| **Overshoot** | Peak value above setpoint (%) | Step response | <10% (conservative), <25% (aggressive) |
| **Steady-state error** | Final error after transients decay | Long-term measurement | 0% (with integral action) |
| **Gain margin** | dB of gain increase before instability | Bode plot | >6 dB |
| **Phase margin** | Degrees of phase lag before instability | Bode plot | >45° |
| **Disturbance rejection** | Error under step disturbance | Load disturbance test | <5% of setpoint |
