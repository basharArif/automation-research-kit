## Fast-Track (8-12 Weeks)

### Objective
Acquire breadth across automation fundamentals, implement working projects, understand modern practices. Suitable for career switchers, junior engineers seeking rapid deployment readiness.

### Week 1-2: Fundamentals & Python Basics

**Learning Objectives**:
- Understand feedback control principles
- Set up Python development environment
- Learn NumPy, Matplotlib for data processing & visualization
- Complete first control simulation

**Topics**:
1. **Control Theory Intro** (2-3 hours)
   - Feedback principle: setpoint, error, feedback
   - Open-loop vs closed-loop control
   - Stability concept
   - Reading: Åström & Murray Ch. 1-2 (40 pages)

2. **Python & Scientific Computing** (4-5 hours)
   - Python basics: variables, functions, classes
   - NumPy: arrays, linear algebra
   - Matplotlib: plotting, visualization
   - Jupyter Notebook workflows

3. **Hands-On Lab**: Implement temperature controller simulation (setpoint tracking with noise)
   ```python
   # Pseudocode
   temperature = 20  # Current
   setpoint = 25     # Desired
   for step in range(1000):
       error = setpoint - temperature
       control_signal = Kp * error  # Proportional term
       temperature += 0.1 * control_signal + noise
   plot(temperature)
   ```

**Deliverable**: Working Python script demonstrating proportional control with plots

**Resources**:
- Coursera: "Python for Data Science" (10 hours)
- NumPy documentation: numpy.org/doc
- Åström & Murray Ch. 1-2: feedback-systems.org

### Week 3-4: PID Control & Tuning

**Learning Objectives**:
- Understand PID controller mathematics
- Tune controllers experimentally
- Analyze stability
- Deploy on simulated systems

**Topics**:
1. **PID Mathematics** (2 hours)
   - Proportional (P): \(u = K_p e\)
   - Integral (I): \(u = K_i \int e dt\)
   - Derivative (D): \(u = K_d de/dt\)
   - Tuning methods: Ziegler-Nichols, loop shaping

2. **Stability Analysis** (1.5 hours)
   - Nyquist criterion
   - Bode plots (frequency domain understanding)
   - Root locus basics

3. **Hands-On Labs**:
   - Lab 1: Tune PID on simulated mass-spring-damper system
   - Lab 2: Anti-windup design (prevent integral saturation)
   - Lab 3: Robustness analysis (step response, load disturbance)

**Deliverable**: Tuned PID controller with performance plots (setpoint tracking, disturbance rejection)

**Resources**:
- Åström & Murray Ch. 6 (PID Control): 30 pages
- "PID Controller Tuning" YouTube series: 5 videos × 20 min
- Open-source: scipy.signal for transfer functions

## Deep-Track (6-12 Months)

### Objective
Develop rigorous theoretical foundations and advanced practical skills. Suitable for researchers, senior engineers entering new domains, PhD students.

### Month 1-2: Rigorous Control Theory

**Learning Objectives**:
- Master state-space methods
- Understand Lyapunov stability rigorously
- Design optimal controllers
- Analyze robustness systematically

**Topics**:

**1. State-Space Methods (2 weeks)**
- State-space representation: \(\dot{x} = Ax + Bu\), \(y = Cx + Du\)
- Controllability and observability (rank conditions)
- System properties from eigenvalues (stability, time constant)
- State feedback design: pole placement
- Observer design: estimate state from limited measurements

**Reading**: Åström & Murray Ch. 3-5 (100 pages) + Ogata "Control Systems Engineering" Ch. 9-10

**Hands-On Labs**:
- Lab 1: Convert transfer function to state-space (and vice versa)
- Lab 2: Design state feedback controller via pole placement
- Lab 3: Design observer; verify observability
- Lab 4: Combined state feedback + observer (output feedback)

**2. Lyapunov Stability (1 week)**
- Lyapunov direct method (construct Lyapunov function)
- Asymptotic vs global stability
- Stability of nonlinear systems
- Robustness margins (gain/phase margins)

**Reading**: Khalil "Nonlinear Systems" Ch. 2-3 (80 pages)

**Hands-On Lab**:
- Lab 1: Prove stability using Lyapunov functions (quadratic form)
- Lab 2: Robust stability analysis under parameter variations

**3. Optimal Control (2 weeks)**
- Linear Quadratic Regulator (LQR): minimize \(J = \int_0^\infty (x^T Q x + u^T R u) dt\)
- Riccati equation derivation and solution
- Model Predictive Control (MPC): receding horizon formulation
- Robust MPC: constraint tightening for uncertainty

**Reading**: Åström & Murray Ch. 7 (60 pages) + Boyd "Convex Optimization" Ch. 9

**Hands-On Labs**:
- Lab 1: Design LQR controller; compare performance vs PID
- Lab 2: Implement MPC using cvxpy or ECOS solver
- Lab 3: Tune Q, R matrices; analyze trade-offs
- Lab 4: Robust MPC with disturbance rejection

**4. Frequency Domain Methods (1 week)**
- Bode plots, Nyquist plots, gain/phase margins
- Loop shaping design
- Compensator design (lead/lag networks)

**Reading**: Åström & Murray Ch. 4 (60 pages)

**Hands-On Lab**:
- Lab 1: Sketch Bode plots by hand; verify with software
- Lab 2: Design compensator for specified margins
- Lab 3: Verify Nyquist criterion for stability

**Month 1-2 Deliverable**: Technical report comparing PID, state feedback, LQR, MPC on benchmark system; derive theoretical insights.

### Month 3-4: Advanced Robotics & ROS2 Deep Dive

**Learning Objectives**:
- Master ROS2 architecture (DDS, QoS, lifecycle)
- Implement SLAM (Simultaneous Localization and Mapping)
- Develop motion planning algorithms
- Understand robot kinematics/dynamics

**Topics**:

**1. ROS2 Advanced (2 weeks)**
- DDS (Data Distribution Service): QoS profiles, real-time guarantees
- Lifecycle nodes: creation → activation → deactivation → cleanup
- Composition: node runtime vs compile-time
- Parameter handling and configuration management
- Testing: pytest + rostest frameworks

**Reading**: ROS2 documentation "Intermediate Concepts" (200+ pages), ROS2 Design Documentation

**Hands-On Labs**:
- Lab 1: Lifecycle node implementation (robot startup sequence)
- Lab 2: Composed nodes (multiple processes)
- Lab 3: QoS experiments (latency vs reliability trade-offs)
- Lab 4: ROS2 testing framework; write unit tests for nodes

**2. Robot Kinematics & Dynamics (2 weeks)**
- Forward kinematics: end-effector position from joint angles
- Inverse kinematics: joint angles from desired end-effector pose
- Jacobian matrix: maps joint velocities to end-effector velocities
- Dynamics: actuator torques from acceleration commands (Lagrangian mechanics)

**Reading**: Siciliano et al. "Robotics: Modelling, Planning and Control" Part 1 & 2 (300 pages)

**Hands-On Labs**:
- Lab 1: Forward kinematics for 3-DOF arm (DH parameters)
- Lab 2: Inverse kinematics (analytical + numerical solutions)
- Lab 3: Jacobian analysis (singularities, workspace)
- Lab 4: Dynamics simulation (joint-space control → Cartesian forces)

**3. SLAM (Simultaneous Localization and Mapping) (2 weeks)**
- Visual odometry: track camera motion from image sequences
- Loop closure detection: identify previously visited locations
- Bundle adjustment: optimize 3D reconstruction
- Implementation: Cartographer (ROS2), ORB-SLAM2

**Reading**: Thrun et al. "Probabilistic Robotics" Ch. 10-12 (100 pages) + recent SLAM surveys

**Hands-On Labs**:
- Lab 1: Monocular SLAM on simulation dataset
- Lab 2: Deploy Cartographer on real robot (TurtleBot3 or similar)
- Lab 3: Analyze loop closure; verify map consistency
- Lab 4: Compare visual + LIDAR odometry

**4. Motion Planning (2 weeks)**
- Graph-based methods: A*, Dijkstra (grid/graph search)
- Sampling-based methods: RRT, RRT* (probabilistically complete)
- Trajectory optimization: quintic polynomials, time-optimal trajectories
- MoveIt framework: integrate planning with ROS2

**Reading**: LaValle "Planning Algorithms" Ch. 2-5 (200 pages) + MoveIt tutorials

**Hands-On Labs**:
- Lab 1: Implement RRT from scratch; benchmark vs RRT*
- Lab 2: MoveIt motion planning for 7-DOF robot arm
- Lab 3: Obstacle avoidance with RRT+collision checking
- Lab 4: Trajectory execution on real/simulated robot

**Month 3-4 Deliverable**: Autonomous navigation demo (SLAM + motion planning; robot maps environment, reaches goal)

### Month 5-6: Machine Learning for Control

**Learning Objectives**:
- Master deep reinforcement learning algorithms
- Implement safe RL with constraints
- Understand imitation learning
- Apply ML to control problems

**Topics**:

**1. Deep Reinforcement Learning (2 weeks)**
- MDPs (Markov Decision Processes): states, actions, rewards, transitions
- Value iteration & policy iteration (discrete, tractable)
- Q-learning (model-free, off-policy)
- Policy gradient methods (PG, A3C)
- Actor-Critic methods (A2C, A3C, PPO)

**Reading**: Sutton & Barto "Reinforcement Learning" Ch. 3-13 (400 pages) + Deep RL survey papers

**Hands-On Labs**:
- Lab 1: Gridworld environment; implement value iteration
- Lab 2: CartPole with Q-learning (discrete approximation)
- Lab 3: Implement PPO from scratch (or use stable-baselines3)
- Lab 4: Continuous control: CartPole + inverted pendulum

**Performance Targets**:
- CartPole-v1: 500 reward (max); PPO achieves this in 50K steps
- MuJoCo benchmark: Compare to published baseline numbers

**2. Safe Reinforcement Learning (2 weeks)**
- Constrained MDPs (CMDPs): optimize reward subject to constraint
- Lagrangian relaxation: dual formulation + gradient-based optimization
- Control Barrier Functions (CBF): formal safety certificates
- Shielding: add safety monitor to learned policy
- Curriculum learning: gradually increase task difficulty

**Reading**: Garcia & Fernández "Safe RL" survey (2015) + recent papers (2020-2024)

**Hands-On Labs**:
- Lab 1: Implement constrained PPO (using Lagrangian relaxation)
- Lab 2: Compare constrained vs unconstrained RL (CartPole with "no flip" constraint)
- Lab 3: Design CBF for simple system; verify safety
- Lab 4: Curriculum learning: start with easy task, gradually increase difficulty

**Key Metric**: Maintain 100% safety constraint satisfaction while maximizing reward

**3. Imitation Learning & Behavior Cloning (1 week)**
- Behavior cloning: learn policy by imitating expert demonstrations
- Dataset aggregation (DAgger): address distribution mismatch
- Inverse reinforcement learning: infer reward function from demonstrations

**Reading**: Torabi et al. "Behavioral Cloning from Observation" (15 pages)

**Hands-On Lab**:
- Lab 1: Collect expert demonstrations (PyGame interface or kinesthetic teaching)
- Lab 2: Train behavior cloning model
- Lab 3: Evaluate and identify failure modes
- Lab 4: Implement DAgger to reduce distribution mismatch

**4. Sim2Real Transfer (2 weeks)**
- Domain randomization: train on randomized simulations
- System identification: estimate real-world parameters
- Reality gap analysis: quantify transfer degradation
- Continual adaptation: online learning on real system

**Reading**: Tobin et al. "Domain Randomization for Sim2Real Transfer" (2017) + recent survey papers

**Hands-On Labs**:
- Lab 1: Train policy in simulation (no randomization); measure transfer failure
- Lab 2: Train with domain randomization; compare transfer success
- Lab 3: Quantify reality gap (measure <3% target)
- Lab 4: Implement online adaptation on real robot

**Month 5-6 Deliverable**: RL-trained controller for simulated/real system with safety guarantees; publish results.

**Robotics Datasets**:
- KITTI: Autonomous driving dataset. 80GB+. Benchmark for 3D object detection.
- nuScenes: Autonomous driving dataset. Larger, more diverse than KITTI.
- RoboNet: Robot interaction dataset. 15M frames. For learning manipulation.

**RPA Datasets**:
- Open datasets limited (proprietary); practice on public websites (with permission)
- Build your own automation workflow on test sites
