# Glossary: Automation Terminology & Definitions

**Version**: 1.0 | **Scope**: 100+ terms from ancient to modern automation

---

## A

**Actuator**: Device that converts control signal (electrical/hydraulic/pneumatic) into physical action (force, displacement, torque). Examples: motors, pneumatic cylinders, solenoids.

**Adaptive Control**: Control system that adjusts parameters online to maintain performance under changing operating conditions or model uncertainty.

**Autonomy Level**: Classification of system independence from human intervention. SAE has L0-L5 for vehicles; ISO 8373 defines robot autonomy.

**Autonomy Quantification Framework (AQF)**: Metrics for measuring degree of autonomy; considers human involvement, system capability, situational awareness.

---

## B

**Behavioral Cloning**: Machine learning technique learning policy by imitating expert demonstrations; simple but suffers from distribution mismatch.

**Bode Plot**: Frequency domain visualization showing magnitude (dB) and phase (degrees) vs frequency (Hz); used for control design and stability analysis.

**Brake (fail-safe)**: Mechanism preventing motion if power is removed; critical for safety in collaborative robots and elevators.

**Bundle Adjustment**: Optimization technique refining 3D reconstruction and camera poses from image matches; core SLAM method.

---

## C

**Chatbot (automation context)**: RPA application automating customer service through pre-defined conversation flows; often rule-based (not neural networks).

**Closed-Loop Control**: System with feedback; actual output compared to desired (reference); error signal drives corrective action. Example: cruise control.

**CMDPs (Constrained Markov Decision Processes)**: Reinforcement learning formulation with optimization objective AND constraint satisfaction; enables safe RL.

**CNN (Convolutional Neural Network)**: Deep learning architecture for vision; learns hierarchical features (edges → textures → objects) through convolution operations.

**Control Barrier Functions (CBF)**: Mathematical certificate proving safety; if barrier function \(h(s) \geq 0\), system state remains safe.

**Control Theory**: Mathematical discipline studying feedback systems, stability, robustness; bridges continuous/discrete, linear/nonlinear systems.

**Controllability**: Property of system; can state reach desired configuration through appropriate control inputs? Rank condition: \(rank([B, AB, ..., A^{n-1}B]) = n\).

**Cyber-Physical System (CPS)**: Tightly integrated computational and physical processes; examples: autonomous vehicles, robotic surgery, smart grids.

---

## D

**Dead Time (Lag)**: Delay between actuation and observable response; creates phase lag complicating control design.

**Deep Learning**: Machine learning using neural networks with 3+ layers; learns representations automatically rather than hand-crafted features.

**Digital Twin**: Virtual replica of physical system; synchronizes with real-world counterpart enabling simulation, optimization, predictive maintenance.

**Degrees of Freedom (DOF)**: Number of independent dimensions of motion. 6-DOF robot arm: 3 positional + 3 rotational; constrains dexterity.

**Domain Randomization**: Training in randomized simulations (physics, rendering, camera pose) to learn robust features generalizing to real world; bridges sim2real gap.

**DQN (Deep Q-Network)**: Pioneering deep RL algorithm; learns state-action values via neural network with experience replay; enabled game-playing AI.

---

## E

**Edge AI**: Deploying machine learning models on edge devices (robots, IoT devices) rather than cloud; reduces latency, improves privacy.

**Eigenvalue**: Characteristic value \(\lambda\) of matrix A; system stability determined by eigenvalues' real parts (negative = stable).

**End-Effector**: Terminal component of robot (gripper, welding torch, sensor); where control effort applied.

**Error**: Difference between desired (reference) and actual (measured) output; feedback control minimizes error.

**Explainability**: Property of AI system; degree to which human understands its decision process. High: linear regression; Low: deep neural networks.

---

## F

**Fail-Safe**: Design principle ensuring system enters safe state upon failure; e.g., power loss → brake engages.

**Feedback**: Measuring output and comparing to reference; core principle of automation enabling self-regulation and stability.

**FMEA (Failure Mode and Effects Analysis)**: Risk assessment method identifying failure modes, causes, effects; basis for safety design.

**Formal Verification**: Mathematical proof that system satisfies specification for ALL possible inputs; contrasts with testing (finite samples). Challenging for complex systems.

**Frequency Domain**: Analysis using sinusoidal inputs and studying output relationships via Fourier/Laplace transforms. Complements time-domain analysis.

---

## G

**Gazebo**: Open-source 3D robot simulation engine; integrates with ROS/ROS2 for rapid prototyping.

**Gradient Descent**: Optimization technique iteratively updating parameters to minimize loss; foundation of deep learning training.

**Gripper**: End-effector for grasping; parallel-jaw (simple), under-actuated (compliant), humanoid (complex dexterity).

---

## H

**Hardware-in-the-Loop (HIL)**: Simulation testing involving actual hardware controller interacting with simulated plant; validates control implementation.

**Homeostasis**: Biological principle maintaining internal equilibrium despite external perturbations through feedback loops; inspires automation design.

**Human-in-the-Loop**: Design where human provides oversight, intervention, or learning to automated system; balances autonomy and control.

---

## I

**IEC 61508**: Functional safety standard for electrical/electronic systems; defines Safety Integrity Levels (SIL 1-4) based on risk.

**IEC 61131-3**: International standard for PLC programming languages (ladder logic, structured text, function blocks).

**IEC 62443**: Industrial cybersecurity standard; defines security levels (SL) 1-4 for automation systems.

**IIoT (Industrial Internet of Things)**: Connected sensors/devices in manufacturing; enables predictive maintenance, optimization, digital twins.

**Imitation Learning**: Learning from demonstrations rather than reward signals; faster convergence but limited to demonstrated behaviors.

**Integral Windup**: PID control problem; integral term accumulates large error when output saturated, causing overshoot. Mitigated via anti-windup schemes.

**Inverse Reinforcement Learning**: Learning reward function from expert demonstrations; enables policy inference without explicit reward.

**ISA-95**: Enterprise-to-manufacturing integration standard; defines levels 0 (process) through 5 (enterprise).

**ISO 26262**: Functional safety standard for automotive systems; ASIL ratings (A-D) based on severity, exposure, controllability.

**ISO 10218**: Industrial robot safety standard; covers robot design, operation, programming, maintenance.

---

## J

**Jacobian Matrix**: Maps joint velocities to end-effector velocities; \(v = J(\theta) \dot{\theta}\). Singularities occur when rank deficient.

**Jerk**: Third derivative of position (acceleration change rate); smooth trajectories minimize jerk for comfort and actuator wear.

---

## K

**Kinematics**: Study of motion without considering forces; forward kinematics (joint angles → end-effector pose), inverse (pose → angles).

**Knowledge Distillation**: Training small "student" neural network to mimic large "teacher"; enables edge deployment with performance retention.

---

## L

**Lag (Phase Lag)**: Phase shift in output relative to sinusoidal input; destabilizing for control; compensated via lead networks.

**Laplace Transform**: Mathematical tool converting differential equations to algebraic form; foundation of classical control theory. \(\mathcal{L}(f(t)) = F(s) = \int_0^\infty f(t)e^{-st}dt\)

**Lead Network**: Compensator increasing phase margin; improves stability and response speed.

**Learning Rate**: Hyperparameter controlling step size in gradient descent; too high → divergence, too low → slow convergence.

**LiDAR (Light Detection and Ranging)**: Laser-based sensor generating 3D point clouds; robust in various lighting, weather; used in autonomous vehicles, robots.

**Linear Quadratic Regulator (LQR)**: Optimal control minimizing quadratic cost \(J = \int(x^T Qx + u^T Ru)dt\); analytically solved via Riccati equation.

**Loop Closure**: SLAM concept; detecting when robot revisits previously mapped location, correcting accumulated drift.

**Lyapunov Function**: Mathematical function proving stability; if \(V(x) > 0\) for all \(x \neq 0\) and \(\dot{V}(x) < 0\), system converges to origin.

---

## M

**Machine Learning**: Field where systems learn patterns from data without explicit programming; includes supervised, unsupervised, reinforcement learning.

**MQTT (Message Queuing Telemetry Transport)**: Lightweight pub-sub protocol for IoT/cloud connectivity; complements OPC UA for cloud integration.

**Model Predictive Control (MPC)**: Optimal control predicting next N steps, solving optimization problem, implementing first action, replanning. Robust, handles constraints.

**Modulation (PWM)**: Pulse-width modulation encodes analog value as duty cycle of switching signal; widely used for motor speed control.

**Motor**: Electrical actuator converting electrical energy to mechanical motion; types: DC, AC induction, stepper, brushless (BLDC).

**MuJoCo**: Open-source physics engine; accurate simulation, differentiable enabling gradient-based learning; increasingly popular for RL robotics.

**Multitasking (RTOS)**: Real-time operating system supporting multiple concurrent tasks; essential for complex automation.

---

## N

**Neural Network**: Biologically-inspired computational model; nodes (neurons) connected via weighted edges; trained to approximate functions.

**Noise**: Undesired signal degrading measurements; control must be robust to noise (Kalman filter, low-pass filtering).

**Nyquist Plot**: Frequency domain stability criterion; plots complex transfer function G(jω); encircles -1 point ⟺ system unstable.

---

## O

**Observability**: Can system state be inferred from measurements? Rank condition: \(rank([C; CA; ...; CA^{n-1}]) = n\).

**Observer**: State estimator inferring unmeasured states from measurements; dual to controller.

**OPC UA (OLE for Process Control Unified Architecture)**: Industrial interoperability standard; semantic data models, security, cross-platform. Key enabler of Industry 4.0.

**Open-Loop Control**: No feedback; control signal determined by reference only. Example: timed heating. Susceptible to disturbances.

**Open Source**: Software with publicly available source code; often MIT, Apache, GPL licenses. Enables community contributions, transparency.

**Overshoot**: Transient response exceeding steady-state value; tuning trade-off with settling time.

---

## P

**PID Controller**: Ubiquitous feedback controller combining Proportional, Integral, Derivative terms. \(u = K_p e + K_i \int e + K_d \dot{e}\). Dominates 95% of industrial control.

**Plant**: System being controlled; in control theory, \(\dot{x} = Ax + Bu\).

**Policy (RL)**: Function mapping state to action; deterministic \(\pi(s) = a\) or stochastic \(\pi(a|s)\) (probability distribution).

**PPO (Proximal Policy Optimization)**: State-of-the-art RL algorithm; policy gradient method with clipped objective preventing excessive updates. Sample-efficient, stable.

**Proportional Control**: Output proportional to error: \(u = K_p e\); simple, no overshoot if tuned conservatively, steady-state error if integral term absent.

**Pruning**: Removing low-weight neural network connections; reduces parameters/computation while maintaining accuracy.

---

## Q

**Quantization**: Converting continuous values to discrete (or lower precision). INT8 quantization: 75% memory reduction, 4x speedup vs FP32.

**QoS (Quality of Service)**: ROS2 parameter; trade-off between reliability (guaranteed delivery) and low latency.

---

## R

**Reality Gap**: Sim2real transfer problem; policy trained in simulation degrades on real hardware due to sensing/dynamics mismatch.

**Reference Tracking**: Control objective: follow desired trajectory/setpoint. Contrasts with regulation (maintain at setpoint).

**Reinforcement Learning (RL)**: Learning from interaction; agent receives reward signal, optimizes accumulated reward over time. Model-free learning.

**Reliability**: System ability to function without failure over time; measured via MTBF (mean time between failures).

**ROS/ROS2 (Robot Operating System)**: Open-source middleware for robotics; pub-sub communication, 3000+ packages, de facto research standard.

**RPA (Robotic Process Automation)**: Software bots automating repetitive digital tasks (data entry, invoice processing); 53% enterprise adoption 2024.

**RTOS (Real-Time Operating System)**: OS guaranteeing worst-case latencies; essential for control timing guarantees.

**RRT (Rapidly-Exploring Random Tree)**: Probabilistically complete motion planning algorithm; fast, handles high-dimensional spaces, asymptotically optimal with RRT* variant.

---

## S

**SCADA (Supervisory Control and Data Acquisition)**: Centralized monitoring of geographically distributed automation systems; operator supervises remote facilities.

**Sensor Fusion**: Combining multiple noisy measurements into state estimate; Kalman filter optimal for linear systems, particle filters for nonlinear.

**Settling Time**: Time for response to reach steady state within tolerance band; inversely related to control bandwidth.

**Sim2Real Transfer**: Training controller in simulation, deploying on real system; domain randomization reduces reality gap to <3%.

**Simulation (Control)**: Mathematical model predicting system behavior; enables design validation, tuning, operator training without risking real equipment.

**SLAM (Simultaneous Localization and Mapping)**: Robot inference of own pose AND map of environment from sensor data; enables autonomous navigation.

**Soft Actor-Critic (SAC)**: Deep RL algorithm maximizing reward + entropy (exploration); excellent sample efficiency, state-of-the-art for continuous control.

**Stability**: Property of system; small perturbations don't cause large responses; asymptotic stability → trajectories converge to equilibrium.

**State (Control)**: System variables determining future behavior; e.g., position, velocity, temperature. State-space representation: \(\dot{x} = Ax + Bu\).

**State Machine**: Discrete system with finite states and transitions; FSM fundamental to PLC logic and embedded systems.

**State-Space**: Mathematical representation using state vector \(x(t)\) rather than differential equations; enables matrix algebra, eigenvalue analysis.

**Steady-State Error**: Difference between reference and actual output after transients decay; eliminated via integral action.

---

## T

**Task Space (Cartesian Space)**: Robot coordinates (3D position + orientation) contrasted with joint space (DOF angles).

**Temporal Logic**: Formal language expressing system properties over time; LTL (Linear Temporal Logic), MTL (Metric Temporal Logic).

**Time Constant**: Characteristic time for system response; \(\tau\) in first-order: \(y(t) = y_{ss}(1 - e^{-t/\tau})\).

**Transfer Function**: Ratio of output to input in Laplace domain; \(G(s) = Y(s)/U(s)\). Captures system behavior for control design.

**Trajectory**: Path through state/configuration space over time; includes velocities, accelerations (contrasts with path, which is geometric).

**Transient Response**: System behavior before reaching steady state; characterized by overshoot, settling time, rise time.

---

## U

**Uncertainty (Robustness)**: Model mismatch, parameter variation, disturbances. Robust control maintains stability despite uncertainty bounds.

---

## V

**Vision-Based Control**: Using camera feedback for control; enables manipulation, navigation; challenges: latency, occlusion.

---

## W

**Workspace**: Reachable region for robot end-effector; bounded by joint limits, singular configurations.

---

## X-Z

**YOLO (You Only Look Once)**: Single-stage object detection neural network; real-time speed (~50 FPS), moderate accuracy (~85-90% mAP).

**Ziegler-Nichols Tuning**: PID tuning method inducing oscillation at stability boundary, deriving gain formulas; practical, heuristic.

---

## Cross-References & Relationships

### Control Concepts
- **Open-Loop** ← → **Closed-Loop**
- **Time Domain** ← → **Frequency Domain**
- **Proportional** → **Integral** → **Derivative** (PID)
- **Stability** ← (via Lyapunov) → **Robustness**
- **Control Theory** ← (rooted in) → **Cybernetics**

### Automation Hierarchy
1. **Sensing** (sensors, perception, state estimation)
2. **Decision** (control, planning, learning)
3. **Actuation** (motors, pneumatics, hydraulics)

### Standards & Compliance
- **IEC 61508** (safety) ⊃ **ISO 26262** (automotive) ⊃ **ISO 10218** (robots)
- **ISA-95** (enterprise integration)
- **IEC 62443** (cybersecurity)

### Modern Automation Stack
- **Perception**: Sensors → Sensor Fusion → State Estimate
- **Planning**: Localization → Mapping → Path Planning → Trajectory Optimization
- **Control**: PID (classical) | MPC (modern) | RL (learning-based)
- **Integration**: ROS2, SCADA, OPC UA, Kubernetes
- **Safety**: Formal Verification, CBF, Shielding
